📘 花卷主题接入指南 3 分钟接入到任意 Markdown 应用

这是什么

一套即插即用的 Markdown 视觉主题包，CSS + JS 两个文件。
任何已经把 markdown 渲染成 HTML 的网页/应用，套上后立即变成杂志/小红书/简报/品牌风格 — 不挑框架不挑 markdown 库。

最快接入（3 步）

  1. 点右上角「导出主题包」选主题 → 下载 zip → 解压两个文件到项目
  2. HTML 引入 css + 容器加 class="markdown-body"
  3. JS 里 applyTheme('xhs-study') 一键应用

代码示例

<link rel="stylesheet" href="./huajuan-themes.css">
<div class="markdown-body" id="content">
  <!-- 这里放 marked / markdown-it 渲染输出的 HTML -->
</div>
<script type="module">
  import { applyTheme, applyPalette, applyLayout } from './huajuan-themes.js';
  applyTheme('xhs-study');           // 一键预设
  applyPalette('xhs-finance');       // 只换色
  applyLayout('magazine');           // 只换布局
</script>

两种使用场景

场景推荐用法
普通博客 / 文档 applyTheme(id) 全局一次切，用户选择持久化到 localStorage
AI Agent / 聊天 服务端给每条消息一个 theme_id，前端 applyTheme(id, msgEl) 单消息独立
带 shadcn/Tailwind 导出时勾「命名空间隔离」(默认开启)，所有 token 加 --hj- 前缀不冲突
给 LLM 自动选主题 把主题清单 JSON 喂进 prompt，让 LLM 在输出开头写 <theme>xxx</theme>

给 AI / LLM 的主题协议

主题 id 不能直接写在 markdown 正文里（会被当文字渲染）。用以下两种之一：

推荐 ⭐ 流式 SSE 首帧

event: theme
data: {"theme_id":"briefing-sunset"}

event: text
data: {"delta":"# 今日简报\n\n..."}

历史回放：数据库独立字段

// messages 表加 theme_id 字段
{ "id": 42, "content": "# 今日简报...", "theme_id": "briefing-sunset" }

// 前端渲染
<div ref={el => el && applyTheme(m.theme_id, el)}
     className="markdown-body" />

主题 id 走元数据通道，不在 markdown 正文里写。完整接入代码见导出包的 README.md。

API 总览

  • applyTheme(id, scope?) 切完整主题
  • applyLayout(id, scope?) 只切布局
  • applyPalette(id, scope?) 只切配色
  • setTypography({heading, body}, scope?) 改字体
  • registerTheme(t) / registerPalette(id, p) 注册外部主题/色卡
  • mountThemePicker(el, opts) 挂一个主题选择器组件
  • 数据: themes, palettes, layoutLabels, colorLabels

完整文档在导出包的 README.md 里，给人和 LLM 都能直接看懂。
